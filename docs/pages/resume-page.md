# 简历解析页

- 版本：v1.1
- 日期：2026-03-23
- 路径：`/resume`

## 1. 页面职责

简历解析页负责：

1. 上传简历文件或直接粘贴简历文本。
2. 调用后端解析接口生成 `resume_summary`。
3. 展示解析结果、保存路径与调试输出。

## 2. 当前交互结构

### 输入区

1. 姓名
2. 文件名
3. 文件上传
4. 简历文本编辑区

### 输出区

1. 用户画像摘要
2. 保存路径
3. 调试输出

## 3. 技术实现

### 前端页面

- 文件：`apps/web/app/resume/page.tsx`
- 类型：Next.js App Router client page

### 状态管理

使用本地 `useState` 管理：

1. `resumeText`
2. `resumeUploadFile`
3. `fileStatus`
4. `parsing`
5. `parsed`
6. `result`

### 文件处理

1. 文本类文件：
   - 浏览器直接 `file.text()`
2. 二进制简历：
   - 支持 `pdf/docx`
   - 通过 `FormData` 传给后端

### 请求方式

调用：

- `POST /v1/resume/parse`

请求封装：

- `apiRequest()`

## 4. 页面当前依赖的后端能力

### 解析能力

1. 文本文件直接读取。
2. `docx -> mammoth`
3. `pdf -> Volcengine OCRPdf`
4. 失败时回退本地旧链路。

### 摘要能力

后端会在完成文本提取后继续生成：

- `resume_summary`

## 5. 调试输出说明

调试输出当前展示的是后端返回的完整 JSON。

主要用于观察：

1. 当前解析器是否为 `volcengine` 或 `legacy`
2. 是否启用 OCR
3. 是否触发了 fallback
4. 当前落盘文件路径

### 当前展示规则

1. 长 JSON 自动换行。
2. 长路径允许折行，避免撑破页面。

## 6. 当前已知限制

1. 调试输出默认直接展示全部 JSON，用户侧信息密度偏高。
2. `remote_ocr_preview`、`volcengine_request_id` 等字段主要用于排查。
3. 当前页面没有把“解析器状态”和“调试明细”分成主次展示。

## 7. 后续建议

### 产品化收口

1. 页面主区域只展示：
   - 解析器
   - OCR 状态
   - 结果来源
   - 摘要结果
2. 原始 JSON 放到折叠调试面板中。

### 验收关注点

1. 真实简历上传是否稳定成功。
2. 落盘 markdown 是否可读。
3. 摘要是否反映真实项目经历。
4. 调试信息是否足够支撑排查，但不会干扰正常使用。
