# 前端文件结构整改计划

基于 CONVENTIONS.md 规范，对现有文件布局的排查结果和待办事项。

---

## 问题 1：session 组件散落在全局 components/

`components/interview-session/` 下 14 个文件 + `interview-session-room.tsx` 都是 `/interview/session` 页面专属，不应放在全局共享目录。

迁移清单：

```
components/interview-session-room.tsx       → app/interview/session/_components/interview-session-room.tsx
components/interview-session/Composer.tsx    → app/interview/session/_components/Composer.tsx
components/interview-session/ConversationTranscript.tsx → app/interview/session/_components/ConversationTranscript.tsx
components/interview-session/copy.ts        → app/interview/session/_lib/copy.ts
components/interview-session/types.ts       → app/interview/session/_lib/types.ts
components/interview-session/CurrentQuestionCard.tsx    → app/interview/session/_components/CurrentQuestionCard.tsx
components/interview-session/InterviewCompletionReport.tsx → app/interview/session/_components/InterviewCompletionReport.tsx
components/interview-session/InterviewPanelDrawer.tsx   → app/interview/session/_components/InterviewPanelDrawer.tsx
components/interview-session/InterviewPanelTabs.tsx     → app/interview/session/_components/InterviewPanelTabs.tsx
components/interview-session/InterviewProgressTab.tsx   → app/interview/session/_components/InterviewProgressTab.tsx
components/interview-session/InterviewQuestionTab.tsx   → app/interview/session/_components/InterviewQuestionTab.tsx
components/interview-session/InterviewRetrospectTab.tsx → app/interview/session/_components/InterviewRetrospectTab.tsx
components/interview-session/RetrospectSummary.tsx      → app/interview/session/_components/RetrospectSummary.tsx
components/interview-session/SessionTopBar.tsx          → app/interview/session/_components/SessionTopBar.tsx
components/interview-session/StageStatusBar.tsx         → app/interview/session/_components/StageStatusBar.tsx
```

迁移后更新所有 import 路径，然后删除 `components/interview-session/` 目录。

---

## 问题 2：RuntimeConfigPanel 应下沉

`components/RuntimeConfigPanel.tsx` 只在 interview 页面使用，移到路由段内部：

```
components/RuntimeConfigPanel.tsx → app/interview/_components/RuntimeConfigPanel.tsx
```

---

## 问题 3：resume / bank / practice 缺少组件拆分

这三个页面所有逻辑堆在单个 page.tsx 里，后续迭代时逐步拆分：

- `app/resume/page.tsx` → 拆出 `_components/`、`_hooks/`
- `app/bank/page.tsx` → 拆出 `_components/`、`_hooks/`
- `app/practice/page.tsx` → 拆出 `_components/`

优先级较低，在功能迭代时顺手做即可。

---

## 问题 4：文件命名风格不统一

当前混用了 PascalCase 和 kebab-case：

```
RuntimeConfigPanel.tsx   ← PascalCase
runtime-config.tsx       ← kebab-case
auth-provider.tsx        ← kebab-case
```

建议统一为 kebab-case（Next.js 社区主流），涉及文件：

- `components/RuntimeConfigPanel.tsx` → `runtime-config-panel.tsx`

路由段内 `_components/` 下的组件文件目前是 PascalCase（如 `ExperienceList.tsx`），如果全局统一 kebab-case 则也需要改。二选一，保持一致即可。

---

## 执行顺序建议

1. 先做问题 1（session 组件迁移），改动量最大但纯机械操作
2. 再做问题 2（RuntimeConfigPanel 下沉），顺手的事
3. 问题 3 随功能迭代逐步做
4. 问题 4 命名统一可以最后批量处理
