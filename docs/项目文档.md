## 功能模块方案

### 1. 运行配置

| 项目 | 说明 |
|------|------|
| 入口 | 首页卡片 01 / Navbar 齿轮图标 |
| 前端 | `runtime-config.tsx` Context + `RuntimeConfigPanel.tsx` UI |
| 存储 | localStorage (`fementor.apiBase`, `fementor.llmBaseUrl` 等) |
| 同步 | `syncLlmConfig`: POST `/v1/runtime/llm-config` → 直连 LLM ping → fallback `/v1/runtime/llm-ping` |
| 健康检查 | 30s 轮询 `/health`，三态: checking/ok/error，Navbar 脉冲点 |

**迭代记录**

| 日期 | 变更 |
|------|------|
| 2025-03 | 初版：三阶段 LLM 配置同步 + 健康脉冲点 |

| 项目 | 说明 |
|------|------|

| 路由 | `document-routes.js` |

### 3. 模拟面试

| 项目 | 说明 |
|------|------|
| 入口 | `/interview` → 选择简历/JD/面经 → `/interview/session` |
| 前端 | `interview/page.tsx` 设置页 + `interview-session-room.tsx` 会话主体 |
| 子组件 | `interview-session/` 下 14 个组件 (Composer, Transcript, Progress 等) |
| 后端 | `interview/llm-service.js` 编排, `turn-service.js` 轮次, `context-service.js` 上下文 |
| 流式 | SSE: `stage` → 占位, `token` → 累积, `result` → 定稿 |
| 路由 | `interview-routes.js` |
| 核心能力 | 智能追问、面经召回、阶段评分、复盘报告 |

**迭代记录**

## 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Next.js 15, React, Tailwind CSS v4, TypeScript |
| 后端 | Node.js, Fastify |
| 数据库 | SQLite (主) + PostgreSQL (面经) |
| LLM | 可配置 base URL，SSE 流式调用 |
| 认证 | Clerk (可选) |
| 检索 | Sirchmunk 向量搜索 |
| 爬虫 | 牛客面经爬虫 |
