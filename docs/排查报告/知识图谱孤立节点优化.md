# 知识图谱孤立节点优化

日期：2026-04-02

## 问题

线上 Supabase 14 篇面经生成了 266 个去重知识点节点，大量孤立节点无法形成有效图谱关联。

根因：
1. 骨架 skeleton.json 只有 10 个一级分类（全是技术类），缺少「行为面」「项目」「安全」「网络」等高频分类
2. LLM 每篇面经独立标注，自由造词导致同义碎片化（如「防抖与节流」vs「防抖」vs「防抖作用」）
3. annotation prompt 无词表约束，LLM 每次措辞不一致

## 改动

### 1. 扩充骨架 `apps/api/data/knowledge-graph-skeleton.json`

新增 6 个一级分类：网络、安全、数据库、行为面、项目、AI。补充现有分类子节点（Vue 加 Vue3/组件通信，浏览器加 IndexedDB/虚拟列表等）。

- 一级分类：10 → 16
- 骨架覆盖节点：175 → 215+

### 2. 归一化层 `apps/api/src/experience/knowledge-graph.js`

三层归一化机制：

- **SYNONYMS 精确映射**（~120 条）：`CSRF攻击→CSRF`、`防抖与节流→防抖节流`、`浏览器缓存→HTTP缓存` 等
- **fuzzyMatchSkeleton 子串模糊匹配**：知识点包含骨架节点名时自动归并（长词优先匹配避免误伤）
- **inferOrphanParents 增强**：先查骨架 children 反向索引 → 再查同义词映射 → 最后用 related 权重推断

应用位置：
- `buildCooccurrenceFromStore`：图谱全量构建时归一化
- `updateGraphIncremental`：增量更新时归一化
- `assemblePipelineResult`（cleaner.js）：写入 DB 前归一化

### 3. 源头约束 `apps/api/src/experience/cleaner.js`

annotation prompt 注入骨架词表（212 词 / ~1500 字符），指令从"优先使用"改为"必须从列表中选取，仅当确实没有匹配项时才可新建"。

## 效果

对 14 篇面经执行 reclean 后：

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 去重知识点 | 266 | 197 |
| 高频节点聚合 | 性能优化×8, AI×2 | 性能优化×22, AI×13 |
| 骨架覆盖率 | ~30% | ~65% |

## 后续可优化

- 批量 annotation：一批面经合并送入 LLM，同一上下文内标签自然复用
- 自动骨架扩充：共现频次超阈值的孤立节点自动挂到最相关一级分类
- 动态 parent 调整：数据量上来后根据共现权重变化重新推断节点归属
