# 前端体验优化：抖动修复、计时暂停、数据缓存

日期：2026-04-03

## 1. Experience 页面登录态抖动

问题：Clerk 初始化期间显示"正在同步登录态..."独占一行，加载完成后消失，导致下方内容垂直抖动。

修复：
- 删除独占段落，改为 header 行内的 Loader2 spinner 图标
- 登录状态区域固定 `min-w-[4.5rem]` 避免宽度变化导致左右抖动

文件：`apps/web/app/experience/page.tsx`

## 2. 模拟面试计时暂停

问题：离开面试页面（切标签页、最小化窗口）后计时继续走，回来时显示的时间包含了不在页面的时间。

修复：通过 `document.visibilitychange` 事件监听页面可见性：
- 页面隐藏时停止 `setInterval`，记录隐藏时间点
- 页面恢复时累加暂停时长到 `pausedMsRef`
- `elapsedMs` 计算时减去总暂停时长

文件：`apps/web/components/interview-session-room.tsx`

## 3. 知识图谱 Canvas 响应窗口 Resize

问题：Canvas 尺寸在 `renderGraph` 时固定，窗口从半屏切到全屏时只显示一半。

修复：用 `ResizeObserver` 监听容器尺寸变化，resize 时更新 canvas 尺寸、重置 ctx transform、更新 center force 并轻微重启 simulation。

文件：`apps/web/app/graph/page.tsx`

## 4. 数据缓存

### 模块级缓存（前端）

问题：面经列表、简历库、JD 库使用组件级 `useState`，页面切换时组件卸载数据丢失，重新请求导致闪白。

修复：改为模块级变量缓存，组件重建时从缓存初始化 state，首次为空才显示 loading，后续静默刷新。

文件：
- `apps/web/app/experience/_hooks/use-experience-list.ts`
- `apps/web/app/interview/_hooks/use-resume-panel.ts`
- `apps/web/app/interview/_hooks/use-jd-panel.ts`

### HTTP Cache-Control（API 端）

在 API GET 接口添加 Cache-Control 响应头，浏览器自动缓存，通过 Next.js rewrite proxy 透传。

| 接口 | Cache-Control |
|------|--------------|
| `GET /v1/experiences` | `public, max-age=30, stale-while-revalidate=300` |
| `GET /v1/experiences/:id` | `public, max-age=60, stale-while-revalidate=300` |
| `GET /v1/resume/library` | `private, max-age=15, stale-while-revalidate=120` |
| `GET /v1/jd/library` | `private, max-age=15, stale-while-revalidate=120` |
| `GET /v1/knowledge-graph` | `public, max-age=60, stale-while-revalidate=600` |

面经和知识图谱用 `public`（不含用户数据），简历和 JD 用 `private`（用户私有数据）。`stale-while-revalidate` 让浏览器在缓存过期后先返回旧数据，同时后台请求新数据。

文件：
- `apps/api/src/routes/experience-routes.js`（原生 + Fastify）
- `apps/api/src/routes/document-routes.js`（原生 + Fastify）
- `apps/api/src/routes/system-routes.js`（原生 + Fastify）
