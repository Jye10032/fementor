# 项目架构设计

- 版本：v0.5
- 日期：2026-03-09

## 1. 架构目标

1. 面试驱动题单沉淀，章节练习基于沉淀题单进行。
2. 检索主路径采用 sirchmunk/rga，本地优先。
3. 证据不足或冲突时触发 WebSearch。
4. 评分与复盘必须可追溯（score -> evidence_refs -> question_bank）。

## 2. 分层

1. API 层：接收请求、参数校验、响应结构稳定。
2. 编排层：interview/practice/retrieval/scoring/retrospect。
3. 能力层：LLM、sirchmunk、websearch、memory writer。
4. 存储层：SQLite（MVP 结构化）+ Markdown（长期记忆）+ 文件系统（原文）。

## 3. 核心流程

1. `start session`：创建 interview_session。
2. `turn evaluate`：每轮问答评分，落 interview_turn。
3. `finish session`：结束会话。
4. `retrospect`：聚合简历+上下文+轮次评分，输出 strengths/weaknesses。
5. `promote`：将面试问题沉淀到 question_bank。
6. `practice`：按 chapter/weakness 拉题训练。

## 4. MVP 范围

1. 单用户本地运行。
2. SQLite 持久化（后续可切 PostgreSQL）。
3. 复盘后自动沉淀 question_bank。
4. WebSearch 先做开关占位。
