# PRD 目录结构（MVP）

- 版本：v0.5
- 日期：2026-03-09

## 1. 产品目标

- 帮助前端候选人通过模拟面试沉淀个性化题单，并进行章节化复练。
- 基于用户文档和检索证据做可追溯评分与复盘。

## 2. 核心闭环

1. 模拟面试（初始题单可为空）。
2. 面试结束复盘（简历 + 提问上下文 + 作答表现）。
3. 题单沉淀（从面试问题回流到 question_bank）。
4. 章节练习（按薄弱项与章节复练）。

## 3. 用户故事

1. 作为用户，我可以上传简历并开始一场模拟面试。
2. 作为用户，我可以在面试结束后得到复盘报告与薄弱项。
3. 作为用户，我可以查看本次面试沉淀出的题单并继续章节练习。

## 4. 功能模块

1. 文档管理：上传、解析、来源记录。
2. 模拟面试：会话开始、逐轮问答、会话结束。
3. 复盘与沉淀：会话聚合分析、题单回流、memory 写入。
4. 章节练习：从 question_bank 拉题训练。
5. 检索与证据：本地优先，证据不足触发 websearch。

## 5. 非功能要求

1. 评分/复盘链路必须可追溯（包含证据计数与来源）。
2. 每次复盘至少产出 1 个可执行改进动作。
3. 面试题沉淀后必须带 chapter、difficulty、review_status。
