## interview_question

用途：一场模拟面试的题目队列。会话开始时一次生成，按 `order_no` 顺序提问。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | text | 主键 |
| session_id | text | 关联 interview_session |
| order_no | integer | 队列顺序，从 1 开始 |
| source | text | 题目来源：`resume/doc/llm` |
| question_type | text | 题型：`basic/project/scenario/follow_up` |
| difficulty | text | 难度：`easy/medium/hard` |
| stem | text | 题干 |
| expected_points_json | text | 预期考点数组 |
| resume_anchor | text | 简历命中片段摘要，便于解释为何出这题 |
| source_ref | text | 来源标记：`resume_summary/job_description/llm_generated` |
| status | text | `pending/asked/answered/skipped` |
| created_at | text | 创建时间 |
| updated_at | text | 更新时间 |

### 设计规则

1. `source` 只表示出题来源，不表示评分证据来源。
2. `order_no` 决定默认出题顺序，不允许同一 session 重复。
3. `status=asked` 表示当前题；`answered` 表示已完成；`pending` 表示队列中尚未开始。
4. `expected_points_json` 用于后续追问、评分解释、题单沉淀，不要求前端直接展示。
5. `resume_anchor` 允许为空；当题目来自简历定制时应尽量填写。
6. `source_ref=job_description` 表示题目主要由当前岗位 JD 驱动。

## question_bank

| 字段 | 类型 | 说明 |
|---|---|---|
| id | text | 主键 |
| user_id | text | 用户ID |
| source_session_id | text | 来源会话ID |
| source_turn_id | text | 来源轮次ID |
| source_question_id | text | 来源队列题目ID |
| source_question_type | text | 来源队列题型：`basic/project/scenario/follow_up` |
| source_question_source | text | 来源队列出题来源：`resume/doc/llm` |
| chapter | text | 章节 |
| question | text | 题目正文 |
| difficulty | text | easy/medium/hard |
| tags_json | text | 标签数组 |
| weakness_tag | text | 关联薄弱项 |
| next_review_at | text | 下次复练时间 |
| review_status | text | pending/done |
| created_at | text | 创建时间 |
| updated_at | text | 更新时间 |

### 追问插入规则

1. 当 `interview_turn.score < 70` 或 `weaknesses_json` 非空时，可触发动态追问。
2. 追问会插入到当前题后的 `order_no + 1`，并写入 `interview_question.question_type = follow_up`。
3. 同一道主问题最多插入一条追问，避免队列无限膨胀。
4. `question_bank` 回流时必须保留 `source_question_id/source_question_type/source_question_source`，这样章节练习才能知道题目来自哪一类面试问题。
